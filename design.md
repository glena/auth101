# Presentation Design Guidelines

## Visual Direction
- Keep slides simple, quiet, and focused.
- Use soft background colors, dark readable text, and restrained accents.
- Avoid saturated palettes, dense diagrams, decorative gradients, and large blocks of prose.
- Use whitespace as the primary layout tool.

## Slide Layout
- Every slide fills the full viewport.
- Each slide should have one clear idea.
- Prefer short headings, compact bullets, and concrete examples.
- Use two-column layouts only when comparing related concepts or pairing explanation with an example.
- Keep code and URL examples readable; wrap long lines intentionally.

## Usability
- Support keyboard navigation with left and right arrow keys.
- Preserve focus visibility for controls.
- Keep controls small and consistent so they do not compete with the slide.
- Presenter notes must be available in a separate window and synchronize with the active slide.

## Typography
- Use system fonts.
- Do not scale text with viewport width.
- Use large type for slide titles and moderate type for body content.
- Keep letter spacing normal.

## Density
- If a slide needs more than five bullets, split it.
- If a slide explains more than one endpoint, flow, or security property, split it.
- Put caveats, historical notes, and spec nuance in presenter notes unless the audience must see them.
